## Problem Statement

Los equipos de desarrollo y personas de alto rendimiento usan Jira, GitLab, Linear para *trackear* tareas, pero ninguna herramienta les ayuda a *priorizar* según su misión, visión y principios. El Cuadrante 2 de Covey (importante NO urgente) es donde vive la estrategia, pero no hay herramienta que lo materialice.

Andrés Alonso, CTO de Necta (10+ años en gobierno de Mendoza), recibe decenas de pedidos de organismos. Hoy decide basado en su conocimiento tácito de cada entidad. Cuando se equivoca, pierde tiempo de desarrollo, elige mal colaboradores en licitaciones, o apuesta por tecnologías que nadie en su equipo sabe usar.

## What Makes This Cool

- **Covey en código**: La matriz urgencia/importancia de los 7 hábitos, combinada con impacto/esfuerzo, como motor de priorización
- **De lo personal a lo colectivo**: Empieza con una persona definiendo su misión → visión → principios → cultura → hábitos → objetivos → metas → tareas
- **Visual-first**: Botones grandes, interfaces limpias, drag-drop en cuadrantes, animaciones via Weave
- **Stack completo Figma**: Agent (diseño) → Make (app core) → MCP (datos/conexión) → Weave (visualizaciones)

## Premises

1. El problema real no es "falta de medición", sino falta de alineación entre propósito (visión/misión) y ejecución diaria
2. Andrés (CTO, gobierno) es un usuario real con poder de decisión y un dolor concreto
3. La priorización basada en urgencia × importancia × impacto / esfuerzo es el núcleo diferenciador
4. El producto empieza personal antes de ser colaborativo
5. Figma Make + MCP + Weave + Agent es el stack correcto para el Makeathon

## Recommended Approach: Quadrant Full Suite

Arquitectura del prototipo para el Makeathon:

### Stack
- **Figma Design Agent**: Diseño UI/UX de la interfaz (matrices, formularios, drag-drop)
- **Figma Make**: App funcional con lógica de negocio (CRUD de visión/misión/objetivos/metas/tareas)
- **Figma MCP**: Conexión entre el diseño en Figma y datos externos (Supabase para persistencia)
- **Figma Weave**: Generación de visualizaciones de cuadrantes, animaciones, gráficos de priorización
- **Supabase**: Backend de datos (equipos, objetivos, tareas, usuarios)

### User Flow (MVP)

```
1. Onboarding: Definir VISIÓN, MISIÓN, PRINCIPIOS
2. Cultura: Definir hábitos y valores del equipo/persona
3. Objetivos: Ámbitos → Roles → Objetivos → Metas → Tareas
4. Priorización: Motor urgencia × importancia × impacto / esfuerzo
   → Score automático → Ordenamiento → Matriz visual
5. Metas Semanales: Distribución automática de lo priorizado en la semana
```

### Priorization Engine

```
Score = (Urgencia × Importancia × Impacto) / Esfuerzo

Donde cada variable es 1-5 (escala Likert):
- Urgencia: ¿Qué tan pronto necesita atención?
- Importancia: ¿Qué tan crítico es para la misión/visión?
- Impacto: ¿Cuánto cambio genera resolverlo?
- Esfuerzo: ¿Cuánto trabajo requiere? (inversa: a más esfuerzo, menor score)
```

### Interfaz

- Matriz de 2×2 (Urgencia × Importancia) con drag-drop de objetivos/tareas
- Timeline semanal con metas asignadas automáticamente
- Visualizaciones Weave: score distribuido, cuadrantes animados, evolución temporal
